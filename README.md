# Invoice Approval System

Automated invoice parsing, validation, and approval recommendation engine. Built for production on a 100% free stack.

---

## Architecture

```
PDF Upload → pdfplumber (text) → Groq Llama 3.3 70B (JSON) → Validation Engine → Recommendation
                                                                      ↓
                                                          Supabase (DB + Storage)
                                                                      ↓
                                                       React Dashboard (Vercel)
                                                                      ↓
                                                        Resend Email Notifications
```

## Tech Stack (all free)

| Layer | Tool |
|---|---|
| AI Extraction | Groq API — Llama 3.3 70B |
| PDF Parsing | pdfplumber |
| Backend | FastAPI → Render.com |
| Frontend | React + Vite + Tailwind → Vercel |
| Database | Supabase (PostgreSQL) |
| File Storage | Supabase Storage |
| Email | Resend |
| Uptime | UptimeRobot (prevents Render sleep) |

## Local Development Setup

### Prerequisites
- Python 3.11+
- Node.js 18+
- Accounts: Supabase, Groq, Resend (all free)

### 1. Clone and setup

```bash
git clone <your-repo>
cd PWC_Project
```

### 2. Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# Fill in your keys in .env
```

Run:
```bash
uvicorn app.main:app --reload
# API at http://localhost:8000
# Docs at http://localhost:8000/docs
```

### 3. Frontend

```bash
cd frontend
npm install

cp .env.example .env.local
# VITE_API_URL leave empty for dev (proxy handles it)
```

Run:
```bash
npm run dev
# App at http://localhost:5173
```

### 4. Supabase Setup

1. Create a project at [supabase.com](https://supabase.com)
2. Go to **SQL Editor** → paste and run `supabase/migrations/001_initial_schema.sql`
3. Go to **Storage** → create a bucket named `invoices` → set to **Public**
4. Copy your project URL and service role key into `backend/.env`

## Deployment

### Backend → Render.com

1. Push code to GitHub
2. Go to [render.com](https://render.com) → New Web Service → connect your repo
3. Point to `backend/` folder, use `render.yaml` config
4. Add all environment variables from `.env.example` in Render dashboard
5. Set up [UptimeRobot](https://uptimerobot.com) to ping `https://your-app.onrender.com/health` every 5 minutes

### Frontend → Vercel

1. Go to [vercel.com](https://vercel.com) → Import project from GitHub
2. Set **Root Directory** to `frontend/`
3. Add environment variable: `VITE_API_URL=https://your-backend.onrender.com/api`
4. Deploy

## Rulebook Structure

Rules are stored per `item_category` and `rule_key`. Supported keys:

| Key | Description | Example Value |
|---|---|---|
| `approved_rate_per_mt` | Approved unit rate | `54500` |
| `min_quantity` | Minimum allowed quantity | `5` |
| `max_quantity` | Maximum allowed quantity | `10` |
| `required_quality_grade` | Accepted grades (comma-separated) | `IS2062 E250` |
| `max_tolerance_pct` | Arithmetic tolerance % | `1` |

---

## Validation Logic

Each invoice runs through:

1. **Arithmetic Check** — `unit_rate × quantity == total_value` (within 1% tolerance)
2. **Rate Check** — invoiced rate matches approved rate in active rulebook
3. **Quantity Check** — quantity within min/max bounds
4. **Quality Check** — quality grade matches required grade

Final verdict:
- All checks pass → **RECOMMEND APPROVAL**
- Any error-level check fails → **RECOMMEND REJECTION**
- No applicable rules found → **NEEDS REVIEW**
